# Actual Budget API Package

Public JavaScript API for integrating with Actual Budget programmatically.

## API Design Principles
- **Backward Compatibility**: Never break existing API methods
- **Clear Errors**: Provide helpful error messages with suggested fixes
- **Type Safety**: Export TypeScript definitions for all public methods
- **Documentation**: Every public method needs JSDoc comments

## Common API Patterns
```javascript
// Initialize first
await actualAPI.init({ dataDir: './data' });

// Load a budget
await actualAPI.loadBudget('my-budget-id');

// Query data
const accounts = await actualAPI.getAccounts();
const transactions = await actualAPI.getTransactions('account-id');

// Modify data
await actualAPI.createAccount({ name: 'Checking', type: 'checking' });
await actualAPI.addTransactions('account-id', [...]);

// Always shutdown
await actualAPI.shutdown();
```

## Testing
- All API methods must have integration tests
- Test with real SQLite database (not mocks)
- Test error cases and edge conditions

## Versioning
- Follow semantic versioning strictly
- Document breaking changes in CHANGELOG.md
- Deprecate before removing (minimum 2 major versions)
