# Desktop Client Package

React-based UI for Actual Budget's desktop and web applications.

## UI Development Guidelines

### Component Standards

- **Reusable First**: Check `src/components/common/` before creating new components
- **Styled Components**: Use our theme system from `@actual-app/components/theme`
- **Accessibility**: All interactive elements need proper ARIA labels and keyboard navigation
- **Mobile Responsive**: Components should work on mobile web (check `src/components/mobile/`)

### State Management

- Use React hooks for local state
- Redux for global app state (accounts, budgets, settings)
- Don't duplicate server state - use live queries from loot-core

### Performance

- Virtualize long lists (transactions, categories)
- Lazy load routes and heavy components
- Memoize expensive calculations with `useMemo`/`useCallback`

### File Organization

```
src/
├── components/      # React components
│   ├── budget/     # Budget-specific UI
│   ├── accounts/   # Account management
│   └── common/     # Reusable components
├── hooks/          # Custom React hooks
├── icons/          # SVG icon components
└── style/          # Global styles
```

## Testing UI

- Component tests: `yarn test`
- Visual regression tests: `yarn vrt`
- E2E tests: `yarn e2e`

## Common Components

- `Button`, `Input`, `Select` from `@actual-app/components`
- `Modal` for dialogs
- `Sheet` for spreadsheet-like tables
